### 3. Fixed Bug 2 — Removed auto-redirect after booking
- **File:** `app/dashboard/tutors/[id]/page.tsx`
- **Change:**
  - Removed `const [redirectTimeout, setRedirectTimeout] = useState<NodeJS.Timeout | null>(null)`
  - Removed `useEffect` that cleared timeout on unmount (lines 71–76 original)
  - Removed `setTimeout(() => router.push('/dashboard/classes'), 2500)` block after `setShowToast(true)`
  - Removed `clearTimeout(redirectTimeout)` from "Book Another Session" button handler
  - Added `setTimeout(() => setShowToast(false), 3000)` to auto-dismiss the success toast only (no redirect)
- **Root cause:** `setTimeout` was explicitly calling `router.push('/dashboard/classes')` 2.5s after booking
- **Revert:** Re-add `redirectTimeout` state, the unmount useEffect, the setTimeout redirect block, and the clearTimeout in the button handler; remove the `setTimeout(() => setShowToast(false), 3000)` line

### 4. Fixed Bug 1 (Calendar) — Added pathname dependency for re-fetch
- **File:** `app/dashboard/calendar/page.tsx`
- **Change:**
  - Added `usePathname` to import from `next/navigation`
  - Added `const pathname = usePathname()` variable
  - Changed `useEffect(() => { ... }, [])` → `useEffect(() => { ... }, [pathname])`
- **Root cause:** Empty `[]` dependency meant the calendar only fetched on first-ever mount. On re-navigation, if the component was served from Next.js router cache without full remount, bookings would never refresh.
- **Revert:** Remove `usePathname` import, remove `const pathname = usePathname()`, change dependency back to `[]`
